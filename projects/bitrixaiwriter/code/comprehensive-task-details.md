## 2. Цель проекта

Разработка системы для автоматизации процесса переписывания описаний товаров в интернет-магазине Заказчика (https://www.souz-pribor.ru/) с использованием технологий искусственного интеллекта (ИИ). Система должна обеспечивать создание уникальных текстов с сохранением ключевых характеристик товаров и возможностью массовой обработки через API.

## 3. Объем работ

- Разработка системы автоматизации для обработки 2000-3000 описаний товаров (из общих 17,000 дублей текстов)
- Реализация полного цикла: выгрузка → обработка → проверка → загрузка

## 4. Требования к функциональности

### 4.1. Общие требования

- Автоматизированное переписывание текстов с использованием ИИ-сервиса
- **ВАЖНО**: Главное требование - отсутствие дублирования между сгенерированными текстами (проверка на внешнюю уникальность через сервисы типа Text.ru может быть исключена)
- Сохранение всех ключевых характеристик и преимуществ товаров
- Обеспечение привлекательности и читабельности текста для потенциальных покупателей

### 4.2. Модули системы

#### 4.2.1. Модуль экспорта данных из Битрикс
- Выгрузка товаров через REST API Битрикс
- Фильтрация товаров по заданным критериям
- Сохранение данных в промежуточный формат (CSV/JSON)

#### 4.2.2. Модуль ИИ-генерации текстов
- Интеграция с Claude API (Anthropic)
- Формирование эффективных промптов для сохранения характеристик товаров
- Обработка результатов генерации

#### 4.2.3. Модуль проверки текстов
- Проверка на дублирование между сгенерированными текстами
- Локальное сравнение с ранее сгенерированными текстами
- Система регенерации при обнаружении дублирования

#### 4.2.4. Модуль импорта обратно в Битрикс
- Подготовка обновленных данных
- Импорт через REST API Битрикс
- Верификация успешности импорта

#### 4.2.5. Админ-панель управления
- Интерфейс для мониторинга процесса
- Просмотр и ручное редактирование текстов
- Статистика и аналитика процесса

### 4.3. Интерфейсы и способы взаимодействия

- Веб-интерфейс (админ-панель)
- Командный интерфейс (CLI) для автоматизации и скриптов
- API для интеграции с внешними системами

## 5. Технические требования

### 5.1. Технологический стек
- Язык программирования: PHP, JavaScript
- API CMS Битрикс
- Интеграция с ИИ-сервисом Claude (Anthropic)
- Frontend: HTML/CSS/JavaScript (Bootstrap)

### 5.2. Требования к ИИ-генерации
- Использование Claude API
- Оптимизация промптов для максимальной вариативности
- Сохранение технических характеристик товаров

### 5.3. Требования к проверке текстов
- Алгоритмы сравнения текстов для предотвращения дубликатов
- Определение степени схожести текстов
- Механизм повторной генерации при высокой схожести

### 5.4. Требования к производительности
- Пакетная обработка товаров
- Асинхронные запросы где применимо
- Оптимизация использования API

## 6. Структура CSV-данных

Заказчик предоставляет выгрузку данных в CSV-формате. Структура файлов:
- ID товара
- Название товара
- Описание товара (требующее переписывания)
- Технические характеристики
- Дополнительные параметры (категория, бренд и т.д.)

- Система должна успешно обрабатывать не менее 95% товаров

- Все технические характеристики товаров должны сохраняться
- Интерфейс должен обеспечивать удобный мониторинг и управление процессом
