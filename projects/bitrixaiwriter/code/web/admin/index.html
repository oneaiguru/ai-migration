Справка по системе

Система автоматизации переписывания описаний товаров предназначена для создания уникальных текстов с использованием искусственного интеллекта (Claude API).

Основные возможности системы:

  • Выгрузка данных о товарах из CMS Битрикс
  • Генерация уникальных описаний с сохранением ключевых характеристик
  • Проверка уникальности текстов
  • Загрузка обновленных описаний обратно в CMS
  • Мониторинг и аналитика процесса

Для начала работы с системой выполните следующие шаги:

  1. Настройте параметры интеграции с Битрикс во вкладке "Настройки"
  2. Укажите API ключ для Claude в разделе настроек генерации
  3. Настройте параметры проверки уникальности
  4. Перейдите на вкладку "Панель управления" и нажмите кнопку "Запустить обработку"

После запуска система начнет автоматически обрабатывать товары и генерировать новые описания. Прогресс можно отслеживать на панели управления и в логах.

Вкладка "Управление текстами" позволяет просматривать и редактировать сгенерированные описания товаров:

  • Просмотр текста - Нажмите кнопку с иконкой глаза, чтобы увидеть сравнение оригинального и сгенерированного текста
  • Перегенерация - Используйте кнопку с иконкой стрелок, чтобы запустить повторную генерацию текста
  • Ручное редактирование - Для текстов, которые требуют корректировки (низкая уникальность), доступно ручное редактирование
  • Применение - Нажмите кнопку с галочкой, чтобы подтвердить текст и применить его к товару

Вы можете использовать фильтры для поиска товаров по статусу, уникальности или названию.

Как долго займет обработка всех товаров?
Время обработки зависит от количества товаров и настроек системы. В среднем система обрабатывает около 500-700 товаров в день при стандартных настройках.

Почему некоторые тексты имеют низкую уникальность?
Уникальность может быть низкой по нескольким причинам: слишком короткий исходный текст, много технических характеристик, которые нельзя изменить, или ограничения модели ИИ. Для таких текстов рекомендуется ручная корректировка.

Можно ли временно приостановить обработку?
Да, вы можете приостановить процесс обработки кнопкой "Пауза" на панели управления. Система завершит текущий пакет и приостановит обработку до возобновления.

Как происходит проверка уникальности?
Система использует два метода проверки: локальное сравнение с ранее сгенерированными текстами и проверку через внешний API (Text.ru). Вы можете настроить порог уникальности в настройках.

Что делать, если возникают ошибки при обработке?
Проверьте логи системы для получения информации об ошибках. Наиболее частые причины: проблемы с подключением к API, превышение лимитов запросов, некорректные данные в исходных описаниях.
