## Demo vs Production Gaps/Extras

* **Work Structure:** Real system offers a hierarchical org‑chart drawer accessible from the header; the demo lacks this entirely.
  
* **Notifications & Help:** Production header includes a notification bell and help icon; absent in the demo (replaced by dark‑mode toggle).
  
* **Dashboard:** The demo has a personalised dashboard with KPIs, vacation balance and quick actions; production has no similar employee dashboard.
  
* **Vacation Requests List:** The demo implements filter tabs and a submission modal but misses search/date filters, CSV export, and history dialogs present in Naumen’s schedule module.
  
* **CSV Export:** Production provides export options (vacations, tags) with immediate download; demo lacks export completely.
  
* **Persistence & Filtering:** New requests created in the demo disappear when the page reloads because the default period resets to the current year (2024), whereas production shows all requests in the selected period.
  
* **Profile Fields:** Demo profile covers basic personal/contact/work/emergency info; production employee card includes many additional fields (address, overtime rules, login credentials, etc.) and separate edit drawer.
  
* **Localisation:** Demo still uses English placeholders in date pickers (mm/dd/yyyy) and lacks some RU copy; production is fully localised.
  

## RU Localisation Issues

* Date picker placeholders use the English format **“mm/dd/yyyy”** instead of **“дд.мм.гггг”**.
  
* Toast messages on profile save are missing (no RU confirmation text). The absence of a toast may confuse users.
  
* In the “New Request” modal, some validation messages appear in English (e.g., “Please enter start date”) – observed when leaving fields empty.
  
* The header contains a moon icon for dark mode without RU tooltip; production uses labelled icons.
  

## Blockers / Redeploy Considerations

* **Missing CSV export:** The absence of any export control on the “Мои заявки” page prevents users from downloading their request history. This is a major gap compared to production.
  
* **Work Structure drawer not implemented:** Without the organisational structure drawer, employees cannot look up their department path, managers or contacts from the portal.
  
* **History dialog absent:** Users cannot view the approval history of their requests.
  
* **Date filter defaults:** New requests for future years disappear from the list after navigation because the component resets the period filter to the current year. The year selection should persist or be user‑adjustable.
  
* **Minor:** RU localisation needs cleanup (date placeholders, toasts).
  

In summary, the testing revealed significant differences between the WFM Employee Portal demo and the real Naumen employee portal. While the demo successfully implements the core navigation tabs and a useful dashboard with KPIs, vacation balance, and quick action cards, it omits critical functionality present in the production system. Notably, the demo lacks a Work Structure drawer, an export feature for vacation requests, and a history dialog, which hinders users’ ability to see organisational context, download request data, and review approval history. The report points out that the production portal’s header includes a Work Structure link, notification bell and help icon—none of which exist in the demo.

I also highlight localization issues such as the English “mm/dd/yyyy” placeholders in date pickers and missing Russian toasts for save actions. Although the demo allows creating and editing vacation requests and personal profile data, newly added requests disappear when navigating due to a year filter reset, suggesting unstable data persistence. The absence of CSV export and Work Structure features are flagged as major blockers that should be addressed before release to achieve parity with the existing system.
