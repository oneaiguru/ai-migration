# Real Naumen vs Demo Portals – Detailed Behaviour & Language Parity

This report compares the real Naumen WFM product (logged in via OIDC) with several Vercel‑hosted demos. All checks were run on October 14 2025 in the user’s time zone (America/New_York). The goal was to validate functional and language parity across modules.

## Methodology

1. **Real product navigation.** I logged into the production WFM environment via the OIDC URL and used the top‑level navigation (`Прогнозы`, `Расписание`, `Сотрудники`, `Отчёты`) to explore all modules: Schedule/Graph, Employees, Forecasts, Reports, Requests, Monitoring, Tasks, Events and Vacations. Screenshots captured the UI and served as ground truth for RU copy and workflows.
   
2. **Demo navigation.** Each demo was opened in a separate tab and explored thoroughly. Actions included clicking every available tab, toggle and modal; switching between modes (day/period, forecast/service); submitting forms (e.g. vacation request); and interacting with filters. I noted any English phrases, console errors, missing features or behaviours that differed from the real product.
   
3. **Documentation review.** Manuals and crosswalk files (`CH2_Login_System.md`, `CH3_Employees.md`, `CH4_FORECASTS.pdf`, `CH5_Schedule_*`, `CH6_Reports.md`, plus the manual crosswalks) were consulted to understand expected functionality and terminology. Each finding is anchored to these references.
   
4. **Reporting format.** Tables below summarise the comparison for each portal/module. A Pass means the behaviour and RU copy match the real product (within the limited scope of the demo). A Fail indicates missing functionality, incorrect terminology or UI behaviour that deviates from Naumen’s manual.

## Overall Findings & Recommendations

1. **Unified shell unavailable** – the pilot domain returned 404. Provide a working URL to test Employees/Schedule integration.
   
2. **Manager portal parity** – The new RU build aligns top navigation with the real product and introduces the “Рабочая структура” filter. However, the dashboard and teams pages are extra features with no counterpart in Naumen. The team modal’s close button is unresponsive. Requests and reports need additional categories to match CH5/CH6.
   
3. **Employee portal** – Behaviour is stable and localisation complete, but it implements self‑service functionality absent from Naumen. Decide whether to keep or align with real processes (manager‑driven requests).
   
4. **Analytics dashboard** – Provides additional business KPIs and visualisations not present in the current Naumen slice. Height clamp fix resolves earlier UX issues.
   
5. **Forecasting & analytics** – Trend analysis and manual adjustments replicate CH4 behaviour well; accuracy metrics are conceptually aligned. However, build forecast, set exceptions and absenteeism workflows are missing. Implement these pages with real API integration and exports to reach full parity.
