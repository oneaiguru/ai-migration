Checklist (run on **new deploy first**, then confirm against the real system)
1. **Shell & Timezone selector**
   - Demo: top-right dropdown should list at least «МСК (UTC+03)» and «Челябинск (UTC+05)». Toggle both and confirm header status + bell remain functional.
   - Real system: validate the same selector exists/navigates without errors.
   - Capture `forecasting-timezone-selector.png` if parity confirmed; log any mismatch.
2. **Build Forecast (`/build`)**
   - Queue tree shows loading shimmer, favourites, and child counts. Multi-horizon cards + absenteeism controls render with RU formatting.
   - After changing timezone, date pickers reflect the selected zone; run “Построить прогноз” and confirm the summary banner + toast still appear.
   - Use **Выгрузить прогноз**; ensure CSV filename includes today’s date and RU decimals.
3. **Exceptions (`/exceptions`)**
   - Day/Interval toggle, multi-interval editor, smoothing sliders, template import/export buttons.
   - Applied-template summary updates when switching timezone.
4. **Absenteeism (`/absenteeism`)**
   - Run history table lists recent runs with status badges/time stamps in the current timezone.
   - Template editor allows periodic + single overrides; export CSV contains RU percentages.
5. **Trend Analysis (`/trends`)**
   - Queue gate: charts remain blank until a queue/period is chosen; equal-period selector available.
   - Strategic/Tactical/Operational tables populate with RU headers; timezone affects the period labels.
6. **Accuracy (`/accuracy`)**
   - KPI deck + error analysis toggles show RU numbers (comma decimals).
   - Detail table lists forecast/fact/Δabs/Δ%/absenteeism/lost calls/AHT/SL (RU formatted). Click **Экспорт CSV** and confirm filename `accuracy_YYYY-MM-DD.csv`.
   - Capture `forecasting-accuracy-detail.png` when confirmed.
7. **Manual Adjustments (`/adjustments`)**
   - Inline validation badges, undo/redo counters, Σ summary behave the same under each timezone.
8. **Reports (shell bell)**
   - Trigger a download (e.g., «Т-13 Табель…») and confirm the bell queue shows the request with RU timestamp + expiry. Filenames should use RU date suffixes.
